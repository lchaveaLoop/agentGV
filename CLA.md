## 📖 前言

感谢您对 AgentGV 项目的关注和贡献！

本贡献者许可协议（CLA）明确了您作为贡献者的权利和义务，以及您授予项目维护者的许可权限。

通过向 AgentGV 项目提交任何贡献（包括但不限于代码、文档、设计、建议等），即表示您同意本协议的所有条款。

## 📝 定义

在本协议中：

- **"项目"** 指 AgentGV 及其所有相关仓库、代码、文档和资源
- **"贡献者"** 指向项目提交任何形式贡献的个人或实体
- **"贡献"** 指贡献者提交给项目的任何内容，包括但不限于：
  - 源代码和目标代码
  - 文档和技术写作
  - 设计、图形和用户界面
  - Bug 报告和问题反馈
  - 功能建议和讨论
- **"维护者"** 指项目管理团队和拥有仓库管理权限的人员
- **"许可证"** 指项目采用的 MIT License（或其他指定开源许可证）

## ✨ 贡献者权利

### 1. 署名权

您有权在您贡献的代码或文档中获得署名：

- 您的名字将出现在贡献记录中
- Git 提交历史将保留您的作者信息
- 贡献者名单中将包含您的信息

### 2. 参与权

您有权参与项目的发展和决策：

- 参与 Issue 和 Pull Request 讨论
- 对项目发展方向提出建议
- 参与社区活动和讨论

### 3. 使用权

您与其他用户一样，享有项目许可证赋予的所有权利：

- 自由使用项目代码
- 修改和分发代码
- 将代码用于商业目的
- 创建衍生作品

### 4. 撤回权

在特定条件下，您可以请求撤回贡献：

- 贡献尚未被合并到主分支
- 有正当理由（如包含敏感信息）
- 经维护者评估同意

**注意**: 已合并的贡献通常无法撤回，但可通过后续提交进行修正。

## 📌 贡献者义务

### 1. 原创性保证

您保证您的贡献满足以下要求：

- **原创性**: 贡献为您原创，或您有权提交
- **非侵权**: 不侵犯任何第三方的知识产权
- **合法性**: 不违反任何法律法规

### 2. 披露义务

如贡献包含第三方内容，您必须：

- 明确标注来源和出处
- 确保符合原许可证要求
- 提供必要的授权证明

### 3. 质量要求

您应尽力确保贡献的质量：

- 遵循项目的代码规范
- 通过必要的测试
- 提供清晰的说明文档
- 响应审查意见

### 4. 诚实义务

您应诚实披露相关信息：

- 如与第三方存在利益关系，应予披露
- 如贡献涉及雇主的知识产权，应获得授权
- 不得故意提交恶意代码或后门

### 5. 持续义务

贡献提交后，您应：

- 及时响应维护者的问题
- 协助修复发现的问题
- 参与必要的代码审查

## 附录 A：常见问题

### Q1: 我提交贡献后还能删除吗？

已合并的贡献通常无法删除，因为已成为项目历史的一部分。但如发现严重问题（如安全漏洞），维护者会协助移除或修正。

### Q2: 公司雇员贡献有什么特殊要求？

如您的贡献可能涉及雇主知识产权，请：
1. 确认公司拥有相关权利
2. 获得公司书面授权
3. 在提交时说明情况

### Q3: 我可以提交第三方代码吗？

可以，但必须：
1. 确保符合原许可证
2. 明确标注来源
3. 不包含限制性条款

### Q4: 贡献会被用于商业用途吗？

可能。MIT License 允许商业使用，任何用户（包括项目方）都可将代码用于商业目的。

### Q5: 我保留对贡献的著作权吗？

是的，您保留著作权，但授予了项目广泛的许可使用权。您仍可将代码用于其他项目。
