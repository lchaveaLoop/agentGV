## 🤖 MiniMax 模型介绍

MiniMax 是中国领先的人工智能公司，专注于大语言模型的研发。MiniMax 系列模型以**高性价比**和**快速响应**著称，特别适合日常开发任务。

### MiniMax 模型特点

- ✅ **经济高效**：成本低于同级别 Qwen 模型
- ✅ **快速响应**：优化了推理速度，适合实时任务
- ✅ **代码能力**：具备良好的代码生成和理解能力
- ✅ **通用推理**：支持一般性推理和分析任务

## 🎯 最佳使用场景

### ✅ 适合使用 MiniMax 的场景

| 场景 | 推荐模型 | 理由 |
|------|----------|------|
| 日常代码编写 | M2.5 | 快速、经济 |
| 简单功能开发 | M2.5 | 性价比高 |
| 代码审查 | M2.5 | 足够识别常见问题 |
| 技术文档 | M1 | 具备良好的理解能力 |
| 数据整理 | M2.5 | 快速处理 |
| 简单调研 | M1 | 成本可控 |
| 创意写作 | M2.5 | 快速响应 |

### ❌ 不适合使用 MiniMax 的场景

| 场景 | 推荐替代 | 理由 |
|------|----------|------|
| 系统架构设计 | Qwen3 Max | 需要深度推理 |
| 视觉任务（图片理解） | Qwen3.5 Plus | MiniMax 不支持视觉 |
| 复杂算法设计 | Qwen3.5 Plus | 需要更强的推理 |
| 核心模块开发 | Qwen3.5 Plus | 需要更高的准确性 |
| OCR 文字识别 | Qwen3.5 Plus | MiniMax 不支持视觉 |

## 🚀 如何使用

### 方式 1：MiniMax 优化模式

直接告诉 Router：

```
切换到 MiniMax 优化模式
```

或使用 CLI：

```bash
node .opencode/preference.js set minimax
```

**效果**：
- 简单任务 → 自动使用 **MiniMax M2.5**
- 复杂任务 → 升级到 **MiniMax M1**

### 方式 2：成本优先模式

```
切换到成本优先模式
```

```bash
node .opencode/preference.js set cost
```

### 方式 3：手动配置

编辑 `opencode.json`：

```json
{
  "agent": {
    "agentgv-router": {
      "model": "minimax/MiniMax-M2.5"
    }
  }
}
```

## ⚠️ 重要注意事项

### 1. 视觉任务不支持

**MiniMax 模型不支持视觉功能**：
- ❌ 图片理解
- ❌ OCR 文字识别
- ❌ 截图转代码
- ❌ 图表分析

### 2. 复杂度自动升级

即使使用 MiniMax 优化模式，遇到高复杂度任务会自动升级到 Qwen 系列。

## 🔧 故障排查

### 问题 1：MiniMax 模型不可用

**解决方案**：
```bash
# 检查配置
node .opencode/status.js

# 重新验证配置
npm run validate
```

### 问题 2：任务没有使用 MiniMax

**原因**：任务被识别为高复杂度

**解决方案**：
- 明确说明是简单任务
- 使用 MiniMax 优化模式

### 问题 3：视觉任务失败

**原因**：MiniMax 不支持视觉

**解决方案**：系统会自动提示切换到 Qwen3.5 Plus
