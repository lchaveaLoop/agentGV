# Contributor Covenant Code of Conduct

## 我们的承诺

为了营造一个开放和友好的环境，我们作为贡献者和维护者承诺：无论年龄、体型、可见或不可见的残疾、种族、性别特征、性别认同和表达、经验水平、教育程度、社会经济地位、国籍、个人外貌、种族、宗教或性取向如何，参与我们的社区都将免受骚扰。

## 🌟 行为准则

### 1. 尊重他人

- 使用欢迎和包容的语言
- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

### 2. 建设性交流

- 保持专业和友好的语气
- 避免人身攻击或侮辱性评论
- 在分歧中寻求理解而非胜利
- 承认并纠正自己的错误
- 向他人学习并表达感谢

### 3. 协作精神

- 鼓励并支持新贡献者
- 分享知识和经验
- 认可他人的贡献
- 优先考虑社区目标而非个人利益
- 在困难任务上相互帮助

## 🗣️ 包容性语言指南

### ✅ 推荐用语

| 场景 | 推荐 |
|------|------|
| 称呼群体 | "大家"、"各位"、"团队" |
| 技术术语 | "主/从" → "主/备"、"primary/replica" |
| 性别中性 | "他们"而非"他/她" |
| 能力描述 | "有挑战"而非"困难" |

### ❌ 避免用语

- 歧视性或冒犯性的笑话
- 基于性别、种族、宗教的刻板印象
- 性暗示的语言或图像
- 贬低或侮辱性的词汇
- 公开或私下的骚扰行为

### 写作建议

```
好的示例:
"请各位审查这个 PR"
"这个功能对所有人都有帮助"
"欢迎新贡献者加入"

避免:
"兄弟们来看看这个"
"这功能很简单，谁都能做"
"只有专家才能改这个"
```

## 🚫 不可接受的行为

以下行为在本社区中不可容忍：

### 骚扰行为

- 使用性化的语言或图像
- 不受欢迎的性关注或示好
- 恶意调侃或人身攻击
- 发布他人私人信息（如住址、邮箱）
- 其他不当的性行为

### 歧视行为

- 基于受保护特征的冒犯性评论
- 刻意使用错误的代词或名字
- 贬低某一群体的言论
- 排斥特定群体的行为

### 破坏行为

- 故意破坏项目或讨论
- 持续打断对话
- 威胁或鼓励暴力
- 鼓励上述任何行为

## 📍 责任范围

项目维护者有责任澄清并执行可接受行为的标准，并将采取适当和公平的纠正措施，应对任何他们认为不当、威胁、冒犯或有害的行为。

项目维护者有权且有责任删除、编辑或拒绝不符合本行为准则的评论、提交、代码、Wiki 编辑、Issue 和其他贡献。

### 维护者职责

1. **监督社区活动**
   - 监控 Issue、PR 和讨论
   - 识别潜在问题
   - 及时介入处理

2. **执行准则**
   - 警告违规行为
   - 暂停严重违规者
   - 永久禁止屡教不改者

3. **保密处理**
   - 保护举报者隐私
   - 公正调查投诉
   - 保密处理敏感信息

## ⚖️ 违规处理流程

### 第 1 步：报告

如发现违规行为，请通过以下方式报告：

- **GitHub Issue**（公开）- 适合一般问题
- **电子邮件**（私密）- 适合敏感问题
- **直接联系维护者**

### 第 2 步：评估

维护者将在 **48 小时** 内评估报告：

- 收集相关信息
- 联系相关方了解情况
- 判断违规程度

### 第 3 步：处理

根据违规严重程度采取相应措施：

| 级别 | 行为 | 处理 |
|------|------|------|
| **轻微** | 无意冒犯、初犯 | 私下警告、要求道歉 |
| **中等** | 重复违规、影响较坏 | 公开警告、临时禁言 |
| **严重** | 恶意骚扰、威胁 | 长期禁言、移除贡献 |
| **极严重** | 违法行为、持续骚扰 | 永久禁止、法律追责 |

### 第 4 步：跟进

- 向举报者反馈处理结果
- 监督整改措施执行
- 记录违规案例（保密）

### 申诉流程

如对所受处理有异议：

1. 提交书面申诉说明
2. 由其他维护者审核
3. 7 个工作日内给出答复
4. 最终决定为终局决定
